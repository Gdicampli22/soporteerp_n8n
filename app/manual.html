1. Visión General de Arquitectura

El sistema Nexus ERP Support centraliza la gestión de tickets provenientes de múltiples canales (Web y Email).

  • Frontend: Desarrollado en Python con Streamlit.
    • dashboard.py: Panel de control para agentes (Kanban, métricas).
    • portal_cliente.py: Interfaz pública para carga de incidentes.
  • Backend: Supabase (PostgreSQL). Persistencia de datos y Storage.
  • Automatización: Make. Orquestación de emails bidireccionales.

2. Configuración de Base de Datos (Supabase)

Tabla: tickets

Columna Tipo Descripción
id int8 (PK) ID único interno (Usado para updates).
id_ticket text ID público (Ej: TK-1745...).
usuario_reportante text Nombre del cliente.
email_reportante text Correo para notificaciones.
comentarios text Historial completo del chat (concatenado).
estado text Abierto, Cerrado

Tabla: reportantes (Lista Blanca)

Controla los permisos para crear tickets por email.

Columna Tipo Descripción
email text (PK) Correo autorizado.
nombre text Nombre del cliente/empresa.

3. Automatización (Make.com)

El sistema utiliza dos "Robots" principales.

🤖 Robot 1: Actualización (Respuestas)

Procesa respuestas a tickets existentes.

  1. Trigger: Gmail (Asunto contiene TK-).
  2. Parser: Extrae ID numérico con Regex TK-(\d+).
  3. Filtro de Seguridad: Verifica que el ID exista antes de continuar.
  4. Búsqueda: Supabase busca id_ticket = TK-{{ParserValue}}.
  5. Update: Actualiza el ticket encontrado. Concatena el historial y pone estado Abierto.

🤖 Robot 2: Creación (Nuevos)

Procesa correos nuevos (sin código TK).

  1. Trigger: Gmail (Asunto NO contiene TK-).
  2. Verificación: Busca el email en tabla reportantes.
  3. Router:
    • Si existe: Crea ticket nuevo y notifica ID.
    • Si no existe: Envía rechazo.

4. Manual del Agente (Staff)

Gestión Diaria

  • Kanban: Arrastre las tarjetas para cambiar estados rápidamente.
  • Respuestas: Al responder desde el Dashboard, el sistema envía automáticamente el correo al cliente y guarda la copia en el historial.
⚠️ Advertencia: Nunca modifique manualmente el campo id_ticket desde la base de datos, ya que rompería la sincronización con el correo.

5. Manual del Usuario (Cliente)

Cómo interactuar por Email

Para asegurar que su solicitud sea procesada:

  1. Envíe correos nuevos a soporte@empresa.com.
  2. Si recibe una respuesta nuestra con un código en el asunto (Ej: [TK-12345]), NO BORRE ESE CÓDIGO al responder.
  3. Responda siempre sobre el mismo hilo de conversación.
