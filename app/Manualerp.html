Bienvenido a Nexus ERP System

Plataforma integral de gestión de soporte técnico con Inteligencia Artificial.

Versión Actual: v9.1 (Production Ready)

¿Qué es Nexus?

Nexus es un sistema desacoplado y contenerizado que permite gestionar incidencias de clientes de manera eficiente. Combina:

  • Dashboard Administrativo: Para coordinadores y agentes (Kanban, BI, Chat).
  • Portal de Clientes: Para autogestión, carga de tickets y seguimiento.
  • IA Copilot: Análisis automático de correos y generación de SLAs.

Para Staff

Gestiona tickets, visualiza métricas de rendimiento y responde correos sin salir de la app.

Para Clientes

Un portal moderno para crear reclamos, ver estadísticas y chatear con soporte.

Instalación y Despliegue

Guía paso a paso para levantar el sistema completo usando Docker.

1. Requisitos Previos

  • Docker Desktop instalado.
  • Git (Opcional, para clonar).
  • Credenciales de Supabase y Gmail.

2. Estructura de Archivos

Asegúrate de que tu carpeta se vea así:

/proyecto
├── docker-compose.yml
├── .env
├── app/
│   ├── dashboard.py
│   ├── portal_cliente.py
│   ├── requirements.txt
│   ├── Dockerfile
│   └── Dockerfile.portal
└── n8n_data/

3. Archivo docker-compose.yml

Este archivo orquesta los 3 servicios (Dashboard, Portal y n8n).

version: '3.8'

services:
  nexus-dashboard:
    build: ./app
    ports: ["8501:8501"]
    env_file: .env
    volumes: [ "./app:/app" ]

  nexus-portal:
    build:
      context: ./app
      dockerfile: Dockerfile.portal
    ports: ["8502:8502"]
    env_file: .env
    volumes: [ "./app:/app" ]

  n8n:
    image: n8nio/n8n:latest
    ports: ["5678:5678"]
    volumes: [ "./n8n_data:/home/node/.n8n" ]

4. Ejecución

Abre la terminal en la carpeta y ejecuta:

docker-compose up -d --build
Acceso a los sistemas:
👮 Dashboard: http://localhost:8501
🏢 Portal Cliente: http://localhost:8502
🤖 Automatización: http://localhost:5678

Configuración Base de Datos

El sistema utiliza Supabase (PostgreSQL). Ejecuta este script en el editor SQL de Supabase para crear las tablas necesarias.

Script SQL Maestro

-- 1. TICKETS
CREATE TABLE public.tickets (
    id_ticket text PRIMARY KEY,
    fecha_creacion timestamptz DEFAULT now(),
    usuario_reportante text,
    email_reportante text,
    titulo text,
    descripcion text,
    estado text DEFAULT 'Abierto',
    prioridad text DEFAULT 'Media',
    agente_soporte text DEFAULT 'Sin Asignar',
    modulo_erp text,
    comentarios text,
    datos_faltantes text,
    adjuntos text
);

-- 2. USUARIOS (STAFF)
CREATE TABLE public.usuarios (
    usuario text PRIMARY KEY, -- Email
    contrasena text NOT NULL,
    nombre_agente text,
    rol text, -- 'Coordinador' o 'Agente'
    activo boolean DEFAULT true
);

-- 3. REPORTANTES (CLIENTES)
CREATE TABLE public.reportantes (
    email text PRIMARY KEY,
    password text NOT NULL,
    nombre text,
    cliente text, -- Empresa
    activo boolean DEFAULT true
);

-- 4. AUDITORIA
CREATE TABLE public.auditoria (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    fecha timestamptz DEFAULT now(),
    usuario_responsable text,
    accion text,
    detalle text,
    entidad text
);

👮 Manual del Agente y Coordinador

Roles del Sistema

  • Coordinador: Tiene visión de "Dios". Ve todos los tickets, todas las métricas y puede gestionar usuarios.
  • Agente: Solo ve sus tickets asignados y su propio rendimiento.

1. El Tablero Kanban

Es la pantalla principal. Los tickets se organizan en columnas:

  • ■ Nuevos: Nadie los ha tocado. Prioridad máxima.
  • ■ En Espera: Ya respondiste, esperando al cliente.
  • ■ En Proceso: Estás trabajando internamente.
  • ■ Finalizados: Problema resuelto.

2. Cómo Responder un Ticket

  1. Haz clic en el botón "Ver" de la tarjeta.
  2. Lee el historial (El mensaje verde es de la IA, el blanco del cliente).
  3. Escribe tu respuesta en el cuadro de texto.
  4. Pulsa "Enviar Respuesta".
Nota: Al enviar, el sistema manda automáticamente un correo al cliente y mueve el ticket a la columna "En Espera".

3. Centro de Reportes (Solo Coordinador)

Desde el menú lateral "Reportes", puedes generar:

  • Rendimiento General: Excel con todos los tickets.
  • Carga de Trabajo: Gráficos de quién trabaja más.
  • Auditoría: Registro de seguridad de movimientos.

🏢 Manual del Portal de Clientes

1. Primeros Pasos

Ingresa a http://tudominio:8502 con tu email corporativo y contraseña.

2. Crear un Reclamo

Ve a la pestaña "Nuevo Reclamo". Completa:

  • Módulo: Dónde falló el sistema (Ventas, Stock, etc.).
  • Asunto y Descripción: Sé detallado.
  • Adjunto: Sube una captura de pantalla del error.

3. Seguimiento y Chat

En la pestaña "Mis Tickets":

  • Selecciona tu ticket en el buscador.
  • Verás el chat estilo WhatsApp con el soporte.
  • Puedes responder directamente ahí.
Si el ticket está en estado "Esperando Cliente", significa que el soporte te hizo una pregunta y necesita tu respuesta para continuar.

🔄 Ciclo de Vida del Ticket

Entender los estados es clave para la automatización.

Estado Significado Acción que lo activa
Abierto Ticket nuevo o cliente respondió. Requiere atención. Email entrante (n8n).
En Revisión El cliente contestó una pregunta del soporte. Respuesta de cliente sobre ticket existente.
Esperando Cliente La pelota está del lado del cliente. Agente envía respuesta desde Dashboard.
Resuelto Problema solucionado. Manual (Agente cierra el caso).
